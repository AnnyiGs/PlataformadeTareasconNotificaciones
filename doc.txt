REQUISITOS DE ARQUITECTURA DE MICROSERVICIOS - ESTADO DE CUMPLIMIENTO
═══════════════════════════════════════════════════════════════════════════════════

✅ CUMPLIDO:

[✓] División en Servicios Independientes
    └─ Tres microservicios implementados:
       • auth-service (autenticación y usuarios)
       • task-service (gestión de tareas)
       • notification-service (notificaciones)
    └─ Cada uno con responsabilidad única y bien definida
    └─ Interfaces REST clara entre servicios

[✓] Contenerización con Docker
    └─ Cada servicio con Dockerfile propio (python:3.11-slim)
    └─ Todos empaquetados en contenedores independientes
    └─ Corriendo exitosamente: 7 contenedores activos
    └─ Verificado: docker compose ps muestra estado Healthy

[✓] Comunicación entre Microservicios
    └─ REST API implementada (HTTP JSON)
    └─ task-service → notification-service (POST /notify)
    └─ Headers personalizados (X-User-Id)
    └─ Flujo E2E verificado y funcional
    └─ Ejemplo: Crear tarea → Notificación automática

[✓] Bases de Datos Independientes (parcial)
    └─ MySQL 8 para task-service y notification-service
    └─ PostgreSQL 16 para auth-service
    └─ Aislamiento por servicio implementado
    └─ Health checks en MySQL funcionando

[✓] Diseño Escalable (parcial)
    └─ Docker Compose gestiona múltiples contenedores
    └─ Health checks en MySQL (healthy status)
    └─ Servicios con restart: always
    └─ Volúmenes persistentes para bases de datos

[✓] Autenticación/Autorización - JWT Tokens
    └─ PyJWT implementado en todos los servicios
    └─ Tokens JWT con HS256 (HMAC-SHA256)
    └─ Expiración de tokens: 480 minutos (8 horas)
    └─ HTTPBearer security scheme en FastAPI
    └─ Validación de tokens en task-service y notification-service
    └─ Flujo E2E completo:
       • auth/register → JWT access_token
       • auth/login → JWT access_token
       • POST /tasks con Bearer token (validado)
       • GET /notifications con Bearer token (validado)
       • PATCH /notifications/{id}/read con Bearer token (validado)
    └─ User isolation implementada (solo acceso a datos propios)
    └─ Todos los endpoints migramos de X-User-Id a Bearer tokens
